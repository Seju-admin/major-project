# HelioCast-AI: Spatio-Temporal PV Generation Forecasting Engine

Welcome to the **HelioCast-AI** repository! This project is an advanced predictive software driven by Deep Learning (CNN-LSTM) and Machine Learning (Random Forest) architectures. It evaluates meteorological momentum to accurately forecast Solar Photovoltaic (PV) generation, helping mitigate grid instability caused by weather volatility.

## Repository Structure & Key Files

Here is a detailed breakdown of the critical scripts, models, and artifacts that power this forecasting engine:

### Core Application Scripts
* **`app.py`**: The main execution script for the interactive Streamlit frontend. It handles the user interface, fetches live REST API data (Open-Meteo), and renders the real-time PV generation dashboard.
* **`train_model.py`**: The dedicated training script. It contains the data engineering pipeline and the training loops for building the deep learning models from the historical dataset.
* **`solar_logic.py`**: The backbone module containing core mathematical logic, API request handlers, and data preprocessing functions that act as a bridge between the raw data and the AI models.
* **`main.py`**: The alternative entry point or backend execution script for running terminal-based inferences without the Streamlit UI.

### Models & Data Artifacts
* **`solar_cnn_lstm.h5`**: The primary Deep Learning model file. This contains the optimized weights of the Spatio-Temporal 1D-CNN and LSTM hybrid architecture.
* **`solar_rf_model.pkl`**: A secondary Machine Learning model based on the Random Forest algorithm, utilized for comparative baseline metrics. *(Note: If this file exceeds GitHub limits, it is stored externally and should be placed in this root directory).*
* **`scaler_X.pkl`**: The serialized Min-Max Scaler artifact. It compresses incoming live API data into the exact 0.0 - 1.0 mathematical boundaries used during the historical training phase to prevent gradient instability.

### Outputs & Visualizations
* **`model_metrics.json`**: A system-generated JSON file that automatically logs and stores the evaluation metrics (like RMSE, MAE, MAPE, and R2 Score) after model testing.
* **`PV_Comprehensive_Dashboard.html` & `PV_Average_Forecast_Dashboard.html`**: Exported interactive HTML dashboards containing in-depth visual analytics, generated directly from the predictive datasets for offline viewing.
* **`forecast.csv` / `forecast.xlsx`**: The raw exported output files containing the numerical time-series predictions generated by the models.
